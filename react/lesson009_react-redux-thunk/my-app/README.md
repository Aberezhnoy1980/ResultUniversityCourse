# [React Redux](https://react-redux.js.org/)

1. Установка

```shell
npm i react-redux redux
```

* React Redux — это библиотека, позволяющая удобно интегрировать библиотеку Redux с библиотекой React.
* В React Redux не включена библиотека Redux. Обе библиотеки нужно ставить по отдельности.
* Для того, чтобы компоненты могли рендериться при изменении состояния в Redux Store, приложение оборачивается в специальный компонент Provider.
* Для рендера компонентов под капотом React Redux использует React Context.

2. Основные средства react-redux:
* useSelector() — позволяет извлекать данные из состояния хранилища Redux с помощью функции-селектора.
* useStore() — возвращает ссылку на то же хранилище Redux, что было передано в Provider.
* useDispatch() — возвращает ссылку на метод dispatch() из хранилища Redux.

3. Практики react-redux:
* Вынос действий (actions) в константы в один или несколько отдельных файлов позволяет уменьшить вероятность опечаток и облегчает разработку, так как все ссылки на действия будут использовать одну и ту же константу из этого файла.
* Генератор действия (action creator) — это функция, которая создает и возвращает объект действия. Путем передачи параметров в эту функцию, мы можем сконфигурировать создаваемый объект действия, чтобы он содержал необходимые данные и свойства.
* Разделение на несколько подстейтов и несколько редьюсеров позволяет упростить управление состоянием приложения, облегчая масштабирование и поддержку кода приложения.
* combineReducers() принимает объект, значениями которого являются различные редьюсеры, и возвращает одну функцию-редьюсер, которую можно передать в createStore() при создании хранилища. Это позволяет объединить несколько редьюсеров в один.

4. Работа с асинхронным кодом

Установка пакета redux thunk:

```shell
npm i redux-thunk
```

* Функция с side-эффектом — это функция, выполнение которой приводит к изменениям во внешних ресурсах или влияет на другие части программы, находящиеся вне функции. Например, функция, которая изменяет состояние, глобальные переменные, или функция, которая выполняет сетевой запрос, являются примерами функций с side-эффектом.
* Redux middleware — это расширение Redux, позволяющее вставлять дополнительную логику между отправкой действия и его обработкой редьюсером. Используется для работы с асинхронным API, логирования, отслеживания ошибок и других задач.
* Redux Thunk — это middleware для Redux, позволяющее передавать в dispatch() функцию вместо действия. Это позволяет выполнять асинхронные операции в Redux.

![redux-thunk-scheme](src/assets/main_scheme.png)

* Redux DevTools — это инструмент для разработчиков, предназначенный для отладки и мониторинга состояния приложения, использующего Redux. Оно предоставляет из себя расширение для браузера, которое позволяет просматривать историю действий, отслеживать изменения состояния, выполнять откат к предыдущим состояниям и многое другое
* compose() — утилита, встроенная в Redux, которая чаще всего используется для применения нескольких расширителей (enhancers) к хранилищу.

5. Redux Toolkit

* Redux Toolkit устраняет недостатки оригинального Redux и предоставляет ряд преимуществ.
* Недостатки оригинального Redux, которые устраняет Redux Toolkit, включают сложную настройку хранилища, необходимость подключения дополнительных пакетов и boilerplate-код для организации Redux-структуры.
* Redux Toolkit делает описание редьюсеров и действий более простым и лаконичным, но может быть менее прозрачным в плане работы Redux, что требует времени и практики для его освоения.
* Оригинальный Redux все ещё широко используется в проектах, и его знание ценится нанимателями. Отличное понимание Redux позволяет быстро освоить Redux Toolkit.
